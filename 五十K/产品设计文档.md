# 五十K记分器 产品设计文档

## 一、产品概述

五十K是一种四人扑克牌游戏，两两一组进行对战。本记分器用于记录游戏中每把的分数，累积计算各队得分，并管理多局游戏历史。

## 二、核心功能

### 2.1 玩家管理
- 四位玩家分为两队：我方（2人）和对方（2人）
- 首次使用需设置玩家姓名
- 每局结束后可选择保持组队或换队（交换一名玩家）
- 玩家数据持久化存储

### 2.2 记分逻辑

#### 2.2.1 基础规则
- 每把基础分总和为200分（我方得分 + 对方得分 = 200）
- **初始化：双方基础分均为0（等待输入）**

#### 2.2.2 炸弹加分
| 张数 | 分数 |
|------|------|
| 5张  | 200分 |
| 6张  | 500分 |
| 7张  | 1000分 |
| 8张  | 2000分 |
| 9张  | 5000分 |
| 10张 | 10000分 |

#### 2.2.3 游次加分
- 我方1、2游：我方+200分，对方-200分
- 我方1、3游：我方+100分，对方-100分
- 对方1、2游：对方+200分，我方-200分
- 对方1、3游：对方+100分，我方-100分

#### 2.2.4 总分计算
```
每把得分 = 基础分 + 炸弹分 + 游次加分
局总分 = 所有把得分之和（支持负分）
级数 = 四舍五入(总分 / 100)
```

#### 2.2.5 胜负判定
- 级数高者获胜
- 级数相同时比较原始总分

### 2.3 多把记分
- 一局包含多把，每把记录后累积
- 支持删除已记录的把手
- 显示累积列表和把数统计

### 2.4 结算流程
1. 点击"结算本局"完成本局
2. 选择新局方式：
   - 新局(不变)：保持当前组队
   - 换队新局：交换一名玩家

### 2.5 历史记录
- 按局显示历史对局
- 可展开查看每把详情
- 支持按玩家名搜索
- 可删除历史记录

### 2.6 统计数据
- 总局数统计
- 玩家个人胜率统计

## 三、数据结构

### 3.1 游戏状态
```javascript
gameData = {
  ourTeam: { baseScore, bombScore, bombs[], tourBonus, totalScore, level, players[] },
  theirTeam: { baseScore, bombScore, bombs[], tourBonus, totalScore, level, players[] },
  roundNum: 1,           // 当前局数
  handNum: 1,            // 当前把数
  isSettled: false,      // 是否已结算
  accumulated: {
    our: { totalScore, hands[] },
    their: { totalScore, hands[] }
  }
}
```

### 3.2 把手记录
```javascript
hand = {
  baseScore,    // 基础分
  bombScore,    // 炸弹总分
  bombs[],      // 炸弹列表
  tourBonus,    // 游次加分
  totalScore    // 本把总得分（可为负）
}
```

### 3.3 历史记录
```javascript
history = {
  id, timestamp, round, totalHands,
  ourTeam: { players[], totalScore, level, hands[] },
  theirTeam: { players[], totalScore, level, hands[] },
  levelDiff, winner
}
```

## 四、界面设计

### 4.1 底部导航
- 记分：主记分页面
- 历史：历史记录查看
- 统计：数据统计

### 4.2 记分页面
1. 状态栏：局数/把数/结算状态/设置按钮
2. 积分卡：双方分数、级数、玩家名
3. 把手列表：已记录的把手累积
4. 输入区：基础分、炸弹、游次
5. 操作区：记录本把/结算本局/重置

### 4.3 弹窗
- 玩家设置弹窗
- 结算结果弹窗
- 规则说明弹窗

## 五、边界情况处理

### 5.1 分数计算
- 支持负分：游次加分可能导致负分
- 基础分初始化为0，等待输入
- 级数计算使用四舍五入，支持负级数

### 5.2 数据验证
- 结算前必须设置所有玩家
- 结算前提示记录未保存的把手
- 删除操作需确认

### 5.3 持久化
- 本地存储：localStorage
- 存储：玩家名、历史记录、统计数据
- 历史：最多保留100条

## 六、技术实现

### 6.1 技术栈
- 纯HTML + CSS + JavaScript
- 无外部依赖
- 响应式设计，适配移动端

### 6.2 关键算法
- 分数累积：每把记录后累加到总分
- 级数计算：Math.round(totalScore / 100)
- 胜负判断：比较级数差
